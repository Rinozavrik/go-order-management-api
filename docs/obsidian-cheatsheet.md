# Шпаргалка по Obsidian

## Templater

Нужен для создания заметок по готовому шаблону.

Как использовать:

1. Создать пустую заметку.
2. Нажать `Ctrl+P`.
3. Выбрать `Templater: Insert template`.
4. Выбрать нужный шаблон:
   - `concept`;
   - `review`;
   - `decision`.

Когда использовать:

- началась новая тема;
- нужно оформить code review;
- нужно зафиксировать архитектурное решение.

---

## Spaced Repetition

Нужен для интервального повторения материала.

Формат карточки:

```text
Вопрос::Краткий ответ
```

Пример:

```text
Чем Go module отличается от package?::Module задаёт корень зависимостей и import path, а package является единицей организации и компиляции Go-кода.
```

Как повторять:

1. Нажать `Ctrl+P`.
2. Выбрать `Review flashcards from all notes`.
3. Попытаться ответить до показа ответа.
4. Оценить, насколько легко вспомнился ответ.

Рабочий процесс:

- ChatGPT формирует 3–5 карточек;
- Codex добавляет их в конец concept note;
- я повторяю их через Spaced Repetition.

Карточки должны проверять понимание, а не тривиальный синтаксис.

## Dataview

Нужен для автоматического построения списков и таблиц по заметкам.

Пока можно не использовать.

Он станет полезен, когда появится много заметок и нужно будет автоматически показывать:

- изучаемые темы;
- темы с низкой уверенностью;
- последние reviews;
- открытые задачи.

Пример metadata в начале заметки:

```yaml
---
type: concept
status: learning
confidence: 2
---
```

Dataview читает эти поля и строит представления.

## Excalidraw

Нужен для схем.

Как использовать:

1. Нажать `Ctrl+P`.
2. Выбрать `Excalidraw: Create new drawing`.
3. Сохранить схему в `diagrams/`.
4. Вставить её в заметку:

```markdown
![[имя-схемы.excalidraw]]
```

Когда использовать:

- зависимости packages;
- lifecycle HTTP-запроса;
- context cancellation;
- goroutines и channels;
- worker pool;
- graceful shutdown;
- транзакции и outbox.

Не рисовать схему, если тему проще объяснить несколькими предложениями.

---

## Минимальный рабочий процесс

После изучения темы:

1. Создать concept note через Templater.
2. Записать ментальную модель своими словами.
3. Выполнить практическую задачу.
4. После review зафиксировать реальные ошибки.
5. Добавить 3–5 карточек.
6. Повторять карточки через Spaced Repetition.
7. Нарисовать схему только при необходимости.

## Главное правило

Obsidian помогает учить Go, а не становится отдельным предметом обучения.
