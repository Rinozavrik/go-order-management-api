# Учебная панель

## Цель

Научиться самостоятельно проектировать, писать, тестировать, профилировать и разворачивать production-oriented backend-сервисы на Go.

Главная цель — не просто изучить синтаксис языка, а понять идиоматичный подход Go к:

- структуре программ;
- работе с ошибками;
- конкурентности;
- управлению ресурсами;
- HTTP;
- базам данных;
- наблюдаемости;
- тестированию;
- профилированию;
- развертыванию.

## Ближайший следующий этап

Базовый синтаксис Go, packages, visibility, значения, указатели, структуры, методы и первые table-driven tests.

## Правила обучения

1. Сначала понять ментальную модель.
2. Затем сравнить её с Node.js и TypeScript.
3. Разобрать минимальный пример.
4. Самостоятельно выполнить практическую задачу.
5. Провести code review.
6. Исправить ошибки.
7. Зафиксировать выводы.
8. Добавить вопросы для повторения.

## Основные проверки

```bash
gofmt -w .
go test ./...
go vet ./...
```

Для конкурентного кода дополнительно:

```bash
go test -race ./...
```

## Принцип работы с Codex

Codex используется как:

- инструмент анализа репозитория;
- code reviewer;
- помощник при поиске ошибок;
- исполнитель конкретных ограниченных изменений.

Codex не должен автоматически выполнять учебные задачи целиком до самостоятельной попытки.

## Источник истины

Актуальное состояние проекта должно храниться в репозитории:

- код — в исходных файлах;
- правила агента — в `AGENTS.md`;
- текущий этап — в `docs/milestones`;
- прогресс — в этом файле;
- архитектурные решения — в `docs/decisions`;
- история ошибок — в `docs/reviews`.

Чаты ChatGPT и Codex не считаются долговременным источником истины.
